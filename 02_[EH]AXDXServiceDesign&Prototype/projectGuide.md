# AX/DX 창업아이디어 설계 및 프로토타입 — Lean 실험 기반 실습 가이드

## 프로젝트 목적
본 프로젝트는 서비스를 설계하는 과제가 아니라,
**비즈니스 가설을 검증하고 Pivot 여부를 판단하는 실험 프로젝트**이다.

프로토타입은 제품이 아니라 "가설 검증 장치"이다.

---

# 1. Business Model 정의 (BMC 작성)

## 팀은 아래 9개 요소를 작성한다
(간략 버전 허용, 단 핵심 고객/가치제안은 필수 구체화)

| 항목 | 내용 |
|---|---|
| Customer Segment | 누구의 문제인가 |
| Value Proposition | 무엇을 해결하는가 |
| Channel | 어떻게 전달되는가 |
| Customer Relationship | 관계 형성 방식 |
| Revenue Stream | 돈은 어디서 발생하는가 |
| Key Activity | 핵심 행동 |
| Key Resource | 필요한 데이터/기술 |
| Key Partner | 외부 이해관계자 |
| Cost Structure | 주요 비용 |

---

# 2. 핵심 가설 정의 (Hypothesis)
다음 3개 가설을 반드시 작성

## (1) 문제 가설
> 우리는 [고객]이 [상황]에서 [문제]를 겪고 있다고 가정한다.

## (2) 가치 가설
> 우리는 [기능/서비스]가 문제 해결에 충분하다고 가정한다.

## (3) 수익 가설
> 고객은 [조건]에서 비용을 지불할 것이다.

---

# 3. 실패 조건 설계 (가장 중요)
프로토타입은 성공을 확인하는 것이 아니라
**가설이 틀리는 조건을 찾기 위해 만든다**

다음 표 작성

| 가설 | 실패 상황 | 관찰 지표 |
|---|---|---|
| 문제 가설 | 고객이 문제를 느끼지 않음 | 무관심 |
| 가치 가설 | 기능을 사용하지 않음 | 사용 포기 |
| 수익 가설 | 비용 지불 거부 | 전환율 0 |

---

# 4. 프로토타입 설계
목표: 최소 노력으로 가설을 깨보기

허용 방식
- 클릭형 화면
- 역할극
- Wizard of Oz
- 종이 인터페이스
- 수동 서비스 제공

금지
- 기능 완성 경쟁
- 디자인 완성도 경쟁

---

# 5. 사용자 실험
최소 5명 이상 인터뷰/테스트 수행

반드시 포함할 질문
- 실제로 쓸 것 같은가?
- 언제 사용하지 않을 것 같은가?
- 돈을 낼 이유가 있는가?
- 현재 대안은 무엇인가?

---

# 6. Pivot 판단
실험 후 아래 중 하나 선택

| 유형 | 의미 |
|---|---|
| Persevere | 기존 모델 유지 |
| Customer Pivot | 고객 변경 |
| Problem Pivot | 문제 변경 |
| Solution Pivot | 해결 방식 변경 |
| Revenue Pivot | 수익 구조 변경 |
| Channel Pivot | 전달 방식 변경 |

선택 이유를 데이터 기반으로 작성한다.

---

# 7. 최종 결과물

## 제출 구성
1. BMC (초기)
2. 가설 정의
3. 프로토타입 설명
4. 실험 결과
5. Pivot 결정
6. 수정된 BMC
